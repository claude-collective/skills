## Research Methodology

Each skill is researched to determine:

1. **2026 Recommendation Status**: Is this still recommended in 2026?
2. **Alternatives**: What are the alternatives?
3. **Conflicts**: What should be disabled when this is selected?
4. **Recommends**: What should be recommended when this is selected?
5. **Requires**: What dependencies does this have?
6. **Discourages**: What should show a warning when both are selected?

## Research Findings Summary

### Frontend Frameworks

#### React

- **2026 Status**: Still dominant with largest ecosystem and job market
- **Alternatives**: vue, angular, solidjs
- **Conflicts**: vue, angular, solidjs (mutually exclusive)
- **Recommends**: zustand, react-query, vitest, react-hook-form, react-testing-library
- **Notes**: Works with Next.js (best SSR/ISR), mature libraries, enterprise adoption

#### Vue

- **2026 Status**: Strong with Vapor Mode for better performance
- **Alternatives**: react, angular, solidjs
- **Conflicts**: react, angular, solidjs
- **Recommends**: pinia, vee-validate, vue-test-utils, vue-i18n
- **Notes**: Progressive, good for teams wanting approachable DX

#### Angular

- **2026 Status**: Enterprise favorite with Signals (v19+)
- **Alternatives**: react, vue, solidjs
- **Conflicts**: react, vue, solidjs
- **Recommends**: ngrx-signalstore
- **Notes**: Batteries-included, best for large enterprise teams

#### SolidJS

- **2026 Status**: Fastest reactive UI library, growing but smaller ecosystem
- **Alternatives**: react, vue, angular
- **Conflicts**: react, vue, angular
- **Recommends**: vitest, tailwind
- **Notes**: Best performance, React-like syntax, limited job market

### Meta-Frameworks

#### Next.js App Router

- **2026 Status**: Leading React meta-framework, React Server Components
- **Requires**: react
- **Conflicts**: remix, nuxt
- **Recommends**: react-query, zustand, next-intl, shadcn-ui

#### Remix

- **2026 Status**: Excellent for form-heavy, data mutation apps
- **Requires**: react
- **Conflicts**: nextjs-app-router, nuxt
- **Recommends**: zustand, vitest, zod-validation
- **Notes**: Web standards purist, excellent nested routing

#### Nuxt

- **2026 Status**: Best Vue meta-framework
- **Requires**: vue
- **Conflicts**: nextjs-app-router, remix
- **Recommends**: pinia, tailwind

### Client State Management

#### Zustand

- **2026 Status**: Winner for most React projects - simple, performant
- **Requires**: react OR react-native
- **Conflicts**: redux-toolkit, mobx, jotai, pinia, ngrx-signalstore
- **Recommends**: react-query
- **Notes**: 40%+ of React projects, minimal boilerplate

#### Redux Toolkit

- **2026 Status**: Still best for large enterprise multi-team apps
- **Requires**: react OR react-native
- **Conflicts**: zustand, mobx, jotai, pinia, ngrx-signalstore
- **Notes**: Strict patterns, powerful devtools

#### Jotai

- **2026 Status**: Best for complex interdependent state (atomic)
- **Requires**: react
- **Conflicts**: zustand, redux-toolkit, mobx, pinia, ngrx-signalstore
- **Notes**: Excellent with React Suspense

#### MobX

- **2026 Status**: Good for OOP-style reactivity
- **Requires**: react
- **Conflicts**: zustand, redux-toolkit, jotai, pinia, ngrx-signalstore
- **Notes**: Smaller community than Redux/Zustand

#### Pinia

- **2026 Status**: Official Vue state management
- **Requires**: vue
- **Conflicts**: zustand, redux-toolkit, mobx, jotai, ngrx-signalstore

#### NgRx SignalStore

- **2026 Status**: Modern Angular state with Signals
- **Requires**: angular
- **Conflicts**: zustand, redux-toolkit, mobx, jotai, pinia

- **2026 Status**: Best comprehensive solution, rich features
- **Requires**: react OR react-native
- **Conflicts**: swr

#### SWR

### Backend Frameworks

#### Hono

- **2026 Status**: Best for edge/serverless, multi-runtime
- **Conflicts**: express, fastify
- **Recommends**: drizzle, better-auth, zod-validation
- **Notes**: 14KB, runs everywhere (Node, Deno, Bun, Workers)

#### Express

- **2026 Status**: Legacy but stable, massive ecosystem
- **Conflicts**: hono, fastify
- **Notes**: Falls behind at scale, still fine for small apps

#### Fastify

- **2026 Status**: Best for traditional Node.js with performance
- **Conflicts**: hono, express
- **Notes**: Better than Express for Node monoliths

- **Notes**: Schema-first, heavier but better tooling
